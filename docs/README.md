# 자판기

- 페어: 티거 X 위니

## 🖥 데모 페이지

[클릭해서 이동하기 🏃🏻‍♀️](https://daaaayeah.github.io/javascript-vendingmachine/)

## 🎯 기능 구현 목록

### 1. 상품 관리 탭

- 초기 화면

  - [x] 상품 관리 탭이 클릭된다. (UI)
  - [x] 상품 정보를 입력하는 폼을 렌더링 한다. (UI)
  - [x] 상품 현황을 렌더링 한다. (UI)
    - [x] 최초 상품 목록은 비워진 상태이다. (UI, 도메인)

- 상품 추가

  - [x] 상품명, 가격, 수량이 입력된다. (UI)
    - [x] [예외 - 상품명] 상품명이 공백인 경우
    - [x] [예외 - 상품명] 상품명이 공백으로만 이루어진 경우
    - [x] [예외 - 상품명] 상품명이 10글자를 초과하는 경우
    - [x] [예외 - 상품명] 상품명이 이미 존재하는 경우
    - [x] [예외 - 가격] 가격이 공백인 경우
    - [x] [예외 - 가격] 가격이 정수가 아닌 경우
    - [x] [예외 - 가격] 가격이 100원 미만인 경우
    - [x] [예외 - 가격] 가격이 10,000원을 초과하는 경우
    - [x] [예외 - 가격] 가격이 10원으로 나누어떨어지지 않는 경우
    - [x] [예외 - 수량] 수량이 공백인 경우
    - [x] [예외 - 수량] 수량이 정수가 아닌 경우
    - [x] [예외 - 수량] 수량이 1 미만인 경우
    - [x] [예외 - 수량] 수량이 20개를 초과하는 경우
  - [x] 추가 버튼이 클릭된다. (UI)
  - [x] 상품을 추가한다. (도메인)
  - [x] 상품 현황을 리렌더링 한다. (UI)
  - [x] 구매 가능 상품 현황을 리렌더링 한다. (UI)

- 상품 수정

  - [x] 수정 버튼이 클릭된다. (UI)
  - [x] 수정 시 상품명, 가격, 수량 정보 영역 자체를 인풋 영역으로 변경한다. (UI)
  - [x] 수정 후, 확인 버튼이 클릭된다. (UI)
  - [x] 상품 정보를 수정한다. (도메인)
  - [x] 상품 현황을 리렌더링 한다. (UI)
  - [x] 구매 가능 상품 현황을 리렌더링 한다. (UI)

- 상품 삭제

  - [x] 삭제 버튼이 클릭된다. (UI)
  - [x] 삭제 시 confirm을 활용하여 사용자에게 다시 한번 확인한다. (UI)
  - [x] 상품을 삭제한다. (도메인)
  - [x] 상품 현황을 리렌더링 한다. (UI)
  - [x] 구매 가능 상품 현황을 리렌더링 한다. (UI)

### 2. 잔돈 충전 탭

- 초기 화면

  - [x] 잔돈 충전 탭이 클릭된다. (UI)
  - [x] 보유할 금액을 입력하는 폼을 렌더링 한다. (UI)
  - [x] 현재 보유 금액을 렌더링 한다. (UI)
    - [x] 최초 자판기가 보유한 금액은 0원이다. (UI, 도메인)
  - [x] 자판기가 보유한 동전을 렌더링 한다. (UI)
    - [x] 최초 각 동전의 개수는 0개이다. (UI, 도메인)

- 잔돈 충전

  - [x] 보유할 금액이 입력된다. (UI)
    - [x] [예외] 잔돈이 10원으로 나누어 떨어지지 않는 경우
    - [x] [예외] 입력된 금액이 더해지면서, 현재 보유 금액이 100,000원을 초과하는 경우
  - [x] 충전 버튼이 클릭된다. (UI)
  - [x] 현재 보유 금액을 충전한다. (도메인)
  - [x] 현재 보유 금액을 리렌더링 한다. (UI)
  - [x] 자판기 보유 금액만큼의 동전을 무작위로 생성한다. (도메인)
  - [x] 자판기가 보유한 동전을 리렌더링 한다. (UI)

### 3. 상품 구매 탭

- 초기 화면

  - [x] 상품 구매 탭이 클릭된다. (UI)
  - [x] 투입할 금액을 입력하는 폼을 렌더링 한다. (UI)
  - [x] 투입한 금액을 렌더링 한다. (UI)
    - [ ] 최초 충전 금액은 0원이다. (UI, 도메인)
  - [x] 구매 가능 상품 현황을 렌더링 한다. (UI)
  - [x] 잔돈 반환 테이블을 렌더링 한다. (UI)
    - [ ] 최초 반환된 각 동전의 개수는 0개이다. (UI, 도메인)

- 금액 투입

  - [ ] 투입할 금액이 입력된다. (UI)
    - [ ] [예외] 금액이 10원으로 나누어 떨어지지 않는 경우
    - [ ] [예외] 금액이 10,000원을 초과하는 경우
  - [ ] 투입 버튼이 클릭된다. (UI)
  - [ ] snackbar로 금액 투입 성공 여부를 보여준다. (UI)
  - [ ] 투입한 금액을 충전한다. (도메인)
  - [ ] 투입한 금액을 리렌더링 한다. (UI)

- 상품 구매

  - [ ] 구매 버튼이 클릭된다. (UI)
  - [ ] snackbar로 상품 구매 성공 여부를 보여준다. (UI)
  - [ ] 구매한 상품의 수량을 줄인다. (도메인)
  - [ ] 구매 가능 상품 현황을 리렌더링 한다. (UI)
  - [ ] 상품 현황을 리렌더링 한다. (UI)

- 잔돈 반환

  - [ ] 반환 버튼이 클릭된다. (UI)
  - [ ] snackbar로 잔돈 반환 성공 여부를 보여준다. (UI)
  - [ ] 투입한 금액을 줄인다. (도메인)
  - [ ] 투입한 금액을 리렌더링 한다. (UI)
  - [ ] 현재 보유한 최소 개수의 동전으로 잔돈을 돌려준다. (도메인)
  - [ ] 잔돈 반환 테이블을 리렌더링 한다. (UI)

### 4. 회원

- 비로그인 사용자

  - 회원가입

    - [ ] 로그인 버튼이 클릭된다. (UI)

    - [ ] 회원가입 버튼을 렌더링 한다. (UI)

    - [ ] 회원가입 버튼이 클릭된다. (UI)
    - [ ] 회원가입 폼을 렌더링 한다. (UI)

      - [ ] [예외 - 이메일] 이메일이 공백으로만 이루어진 경우

      - [ ] [예외 - 이메일] 이메일이 이미 존재하는 경우
      - [ ] [예외 - 이름] 이름이 공백인 경우

  - 로그인

    - [ ] 로그인 폼을 렌더링 한다. (UI)
    - [ ] 사용자 정보가 입력된다. (UI)
    - [ ] 확인 버튼이 클릭된다. (UI)

- 로그인 사용자

  - 초기 화면

    - [ ] 썸네일을 렌더링 한다. (UI)
      - [ ] 로그인한 유저의 이름 중 첫 번째 글자를 썸네일처럼 만든다.
    - [ ] 관리자 메뉴를 렌더링 한다. (UI)

  - 회원 정보 수정

    - [ ] 회원 정보 수정과 로그아웃 메뉴를 렌더링 한다. (UI)
    - [ ] 회원 정보 수정 메뉴가 클릭된다. (UI)
    - [ ] 회원 정보 수정 폼을 렌더링 한다. (UI)
    - [ ] 확인 버튼이 클릭된다. (UI)

  - 로그아웃

    - [ ] 썸네일이 클릭된다. (UI)
    - [ ] 회원 정보 수정과 로그아웃 메뉴를 렌더링 한다. (UI)
    - [ ] 로그아웃 메뉴가 클릭된다. (UI)

## ✅ 요구 사항

1. 공통

- [x] 도메인 영역을 타입스크립트를 이용하여 구현한다.
  - [x] Interface 또는 type을 이용하여, 주요 도메인 객체의 타입을 정의하고 설계한다.

2. 라우팅 기능

- [x] Browser History Api를 이용하여 SPA처럼 라우팅을 적용한다.
  - [x] 매번 페이지를 로드 하는 것이 아닌, 히스토리를 관리하고, 페이지를 url에 따라 동적으로 렌더링한다.
- [x] 상품 관리, 잔돈 충전, 상품 구매 페이지는 모두 동적으로 렌더링해야 한다.

3. 관리자 회원 기능

- [ ] 관리자의 회원 가입을 위한 기능은 json-server-auth를 이용한다.

4. 테스트

- [x] 비즈니스 로직에 대한 단위 테스트를 Jest로 작성한다.
- [ ] 사용자 입장에 대한 테스트를 Cypress로 작성한다.

5. UI

- [x] figma 시안을 기준으로 구현한다.
